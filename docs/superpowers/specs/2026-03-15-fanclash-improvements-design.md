# FanClash 기존 기능 개선 — 설계 문서

> 작성일: 2026-03-15
> 접근법: UX + 안정화 혼합 (접근법 B)

## 개요

FanClash Phase 1 기능이 대부분 완료된 상태에서, 출시 전 품질을 높이기 위한 개선 작업.
4개 영역을 다룬다: 하입 피처 설정 UI, 대시보드 레이아웃, 연동 UX, 안정화.

---

## 섹션 1: 하입 피처 5종 설정 UI + CSS XSS 수정

### 문제

- 음악/가챠/물리/영토/날씨 위젯은 대시보드에서 설정 변경이 불가능
- `WidgetSettingsModal.tsx`가 818줄 모놀리식 — 유지보수 어려움
- Custom CSS가 `dangerouslySetInnerHTML`로 삽입, XSS 위험

### 설계

#### 위젯별 설정 항목

| 위젯 | 설정 항목 | 기본값 |
|------|----------|--------|
| 음악 | 볼륨 (0-100), 시각화 ON/OFF, 스케일 타입 (펜타토닉/메이저/마이너) | 70, ON, 펜타토닉 |
| 가챠 | 히스토리 표시 ON/OFF, 최대 히스토리 수 (5/10/20), 최소 금액 | ON, 10, 1000원 |
| 물리 | 최대 오브젝트 수 (20/50/100), 중력 강도 (약/중/강), 이모지 크기 | 50, 중, 자동 |
| 영토 | 격자 크기 (10x6/20x12/30x18), 리더보드 표시 ON/OFF, 점령 최소 금액 | 20x12, ON, 1000원 |
| 날씨 | 파티클 밀도 (낮/중/높), 날씨 변경 기준 시간 (3/5/10분), 화면 흔들림 ON/OFF | 중, 5분, ON |

#### 컴포넌트 분리

`WidgetSettingsModal.tsx`를 분리:

```
src/components/dashboard/settings/
├── AlertSettings.tsx
├── RankingSettings.tsx
├── GoalSettings.tsx
├── AffinitySettings.tsx
├── BattleSettings.tsx
├── TeamBattleSettings.tsx
├── TimerSettings.tsx
├── MessagesSettings.tsx
├── RouletteSettings.tsx
├── MusicSettings.tsx      ← 신규
├── GachaSettings.tsx      ← 신규
├── PhysicsSettings.tsx    ← 신규
├── TerritorySettings.tsx  ← 신규
├── WeatherSettings.tsx    ← 신규
└── StylePresets.tsx
```

`WidgetSettingsModal.tsx`는 위젯 타입에 따라 해당 설정 컴포넌트를 동적 로드하는 셸 역할만 담당.

`STYLE_PRESETS` 상수 배열은 `StylePresets.tsx`로 함께 이동한다.

#### CSS XSS 수정

- `dangerouslySetInnerHTML` 제거
- CSS 새니타이저 함수 구현: `sanitizeCSS(raw: string): string`
  - `javascript:`, `expression()`, `@import`, `url()` 내 `data:` 제거
  - 허용된 CSS 속성 화이트리스트 기반 필터링
- 오버레이에서 `<style>` 태그를 DOM API로 안전하게 삽입

## 섹션 2: 대시보드 레이아웃 개편

### 문제

- 8개 섹션 수직 스택 → 과도한 스크롤
- Quick Actions가 사이드바와 100% 중복
- 무료 Stats 페이지가 거의 빈 화면

### 설계

#### 레이아웃 변경

**Before:**
```
[ConnectionStatus]
[Pro 업그레이드 배너]
[퀵 스탯 4개]
[온보딩 가이드]
[최근 도네이션]
[탑 팬]
[활성 위젯]
[퀵 액션 5개]
```

**After:**
```
[ConnectionStatus + 퀵 스탯 4개]     ← 한 줄로 병합
[온보딩 가이드]                       ← 완료 시 자동 숨김 (다시 펼치기 가능)
[2컬럼 레이아웃]
  좌: 최근 도네이션 (실시간)
  우: 탑 팬 + 활성 위젯 요약
[Pro 업그레이드]                      ← 하단 서브틀 배너
```

#### 변경 목록

1. **Quick Actions 섹션 제거** — 사이드바와 중복
2. **온보딩 가이드 자동 숨김** — 4단계 완료 시 접힘, 재펼침 가능 (접힘 상태는 `localStorage`에 저장)
3. **퀵 스탯을 헤더 영역에 통합** — ConnectionStatus 옆에 컴팩트 배치
4. **2컬럼 레이아웃** — 데스크탑에서 한 화면에 핵심 정보 (모바일은 스택)
5. **Pro 배너 톤다운** — 상단 강조 → 하단 서브틀 배너

#### 무료 Stats 페이지

- 무료에서도 표시: 총 도네이션 수, 유니크 팬 수, 오늘 도네이션 수
- Pro 게이팅 유지: 기간 필터, 차트, 시즌, CSV 내보내기

## 섹션 3: 연동 UX 개선

### 문제

- 라이브 필수 플랫폼(TikTok/Chzzk/Soop) 오프라인 시 무피드백
- 연결 타임아웃 없음 (무한 대기 가능)
- TikTok만 자동 재시도, 나머지 수동
- 에러 메시지가 기술적이고 영어

### 설계

#### 라이브 필수 표시 + 사전 체크

- 카드에 "LIVE 필수" 뱃지 상시 표시
- 연결 클릭 시 라이브 상태 사전 체크 (Chzzk/Soop은 API, TikTok은 연결 시도 후)
- 오프라인이면: "방송을 먼저 시작해주세요" 안내 표시

#### 연결 타임아웃 + 상태 피드백

타임아웃은 2단계로 구분:
- **플랫폼 커넥터 타임아웃 (15초)**: 각 플랫폼 API/WebSocket 연결 시도의 최대 대기 시간. 기존 Socket.IO 클라이언트의 `timeout: 10000`(프론트-서버 간)과는 별개.
- **프론트엔드 UX 타임아웃**: 서버로부터 성공/실패 응답이 없으면 15초 후 타임아웃 메시지 표시.

동작:
- 연결 시도: 실제 완료까지 스피너 유지 (최대 15초)
- 성공: "✓ 연결됨"
- 실패: 상황별 한국어 메시지
- 타임아웃: "플랫폼 서버가 응답하지 않습니다"

#### 통일된 자동 재시도

- 모든 플랫폼: 30초 간격, 최대 5회
- 재시도 중 카드에 "재연결 중... (2/5)" 표시
- 전부 실패 시: "자동 재연결 실패. 수동으로 다시 시도해주세요."

#### 에러 메시지 한국어화

| 상황 | 메시지 |
|------|--------|
| 키/토큰 오류 | "인증 키가 올바르지 않습니다. 다시 확인해주세요." |
| 방송 오프라인 | "방송이 꺼져 있습니다. 방송 시작 후 다시 시도해주세요." |
| 네트워크 | "네트워크 연결을 확인해주세요." |
| 타임아웃 | "플랫폼 서버가 응답하지 않습니다. 잠시 후 다시 시도해주세요." |

#### DB 상태 동기화

- 서버 측 30초 간격 heartbeat로 실제 연결 상태 확인
- 연결 끊김 시 DB `connected: false` 자동 업데이트
- 프론트엔드 폴링 제거 → Socket.IO 이벤트 기반 전환

## 섹션 4: 추가 안정화

### 4.1 DonationPhysics 메모리 누수 수정

- 언마운트 시 `Matter.Engine.clear()` + `Matter.World.clear()` 명시적 호출
- `bodiesMetaRef` 맵 클리어
- `requestAnimationFrame` 루프 정리 보장
- `triggerDrop` 내 `setTimeout` ID를 추적하여 언마운트 시 전부 `clearTimeout` 호출 (stale ref 방지)

### 4.2 파괴적 액션 확인 모달

공통 `ConfirmModal` 컴포넌트:

| 액션 | 확인 메시지 |
|------|------------|
| 위젯 삭제 | "이 위젯을 삭제하시겠습니까?" |
| 목표 리셋 | "목표 진행도가 초기화됩니다" |
| 계정 삭제 | 셀프서비스 + 2단계 확인 (비밀번호 입력) |
| 연동 해제 | "연결이 끊어집니다" |

### 4.3 폼 검증 (Zod)

- 위젯 설정: 금액 양수, 시간 범위 내, 필수값 체크
- 연동 설정: 키/토큰 포맷 사전 검증
- 프론트 + 서버 동일 스키마 적용

### 4.4 에러 바운더리 강화

- 대시보드 각 섹션별 ErrorBoundary 추가
- 한 섹션 실패해도 나머지 정상 표시
- 실패 섹션에 "다시 시도" 버튼

### 4.5 접근성 기본

- 토글 버튼: `aria-label`, `aria-pressed`
- 모달: 포커스 트랩 + ESC 닫기 통일
- 연동 상태: 색상 + 텍스트 병행 (색각 이상 대응)
